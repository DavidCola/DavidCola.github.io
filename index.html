Bink!

Bink is a 2-D arcade style game where you play as a little fella and jump from node to node avoiding hazards, scoring points, and racing against an ever-shortening death timer. This was my senior game project at Digipen that I worked on with my team Bonsai Beach. I was the producer on the game and additionally as the primary gameplay programmer. During my time on this project, I learned a lot about team and task management, as-well-as how to polish out a game to make it feel slick. This project is one of my favorites because of how smooth and polished it is. From the offset of development, one of our goals for Bink! was to make it feel like a completely polished experience despite it’s small size.

GutterBall

Gutterball is a 3d third person action game where you must fight through a twisted bowling ally fighting off evil pins that have come to life with your weapon of choice, the Bowling Ball! This was my junior student project that I worked on with my team Midnight Hour. On this project I operated as the primary technical as well as being the head of quality assurance for the team. As the primary technical designer, I was responsible for collaborating with designers to implement gameplay features into the unreal engine using c++. The system I was responsible for implementing included the combat system, player controller, enemy behaviors, combo system, as well as a lot of other more minor tasks. My favorite part of this project was learning how to utilize the unreal engine in a productive manner. I learned a lot about the engine and it structure, as well as how to extend and utilize its functionality for my own purposes. I also really enjoyed working with designers on this project, collaboration is one of my favorite parts of game development, and for this project I got to work with some awesome designers. As head of quality assurance for the project I updated a weekly QA checklist and made sure a updated build of the game was created and tested each week. I learned a lot about how to best keep watch for bugs and problems when working with a large team and having a load of updates and mergers 5each week.

Lost In Transit

Lost in Transit is a 2-D platformer game in which you play as a digital mailman trying to navigate corrupted file systems to deliver an email. You have to use abilities to dash and teleport your way through the level while avoiding hazards along the way. This was my sophomore project that I worked on at DigiPen with team “Midnight Hour” as a systems programmer in a custom engine. The big thing about this game is that for the class we had to build it entirely in a c++ custom engine, so I was part of the programming team that made that happen. I was specifically responsible for a lot of subsystems that needed to load in data such as textures, animations, sounds, and input settings. On this team we also had designs that needed to work within that custom engine, so a big part of my work was creating json based config files for designers to be able to add new content using my systems. This project was especially fun and interdesting for me because we got to create and design a custom engine ourselves. While working with engines like unreal and unity is convenient, I found the ability to be flexible and design systems very specifically to meet our goals a very rewarding experience.

Untitled Square Game

Untitled Square Game is a couch co-op platformer party game where you race other players up a vertical play space. This game was a single semester freshman project that I worked on at DigiPen with team “Hint of Death”. On this project, I worked as one of many programmers and I was responsible for systems such as input and particles. The biggest thing I learned on this project was how NOT to create a input system, while in the end I made it work, it admittedly was not very clean or efficient code. It was still it an incredibly valuable experience because I got to take what I learned from this project and apply it to my next project “Lost In Transit” where I was able to create a much better and snazzier event driven input system. I also learned a lot about how to design and get a couch co-op multiplayer system to work in terms of its UI.

Pressure

Pressure is a 2-D wave-based bullet hell where you constantly have to switch between moving and shooting in a frantic battle against anomalous alien forces. This was one of my earliest solo projects that I worked on before attending DigiPen when I was just getting into game development for the first time and one of my first experiences using the unity engine. This project holds a special place in my heart as it was the first game I developed through to a finished product. I had a blast working on it from designing the combat, creating the pixel art, and doing the sound balancing. This game not only helped me find my passion for game development, but also helped me learn about how all the different elements of games like sounds, art, and gameplay must come together to create a fun and compelling experience. It also helped me learn important coding principles, such as component-based behaviors and was just good practice for learning c# and how to use the unity system effectively. This is a game I still come back to just to play and that I genuinely enjoy playing. It is near the top of my bucket list to remake now that I have learned so much more about how to create and design a game.

Tanks The Game

Tanks the Game is a 3d split screen party game where you and up to 4 friends fight each other in tanks in various maps. This game was another solo project I worked on around 2018 and was created in the unity engine. I fully programed and designed this game. One of the most challenging parts of this game was to create the 4 player spit screen. Looking back at that code I is extremely scuffed, and programmed in one of the worst possible way, honestly considering I was teaching myself to code at the time I am surprised I got it working at all. My favorite part of this game had to be designing the maps, I have about 12 maps in the game many of witch have their own gimmicks and stage hazards to make them distink. Again I would love to come back to this game and remake it with my current knowledge.
